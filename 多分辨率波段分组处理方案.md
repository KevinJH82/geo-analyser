# 高精度多分辨率波段分组处理方案

## Context

用户追求靶区定位高精度，不接受重采样带来的混合像元问题。
各传感器存在多种原始分辨率，需要**按分辨率分组独立处理、独立输出**，
后续在 GIS 中叠合，完全保留各波段的原始像元精度。

---

## 传感器波段分组规则

| 传感器 | 分组名 | 波段 | 分辨率 | 参与处理 |
|--------|--------|------|--------|---------|
| Landsat 8/9 | optical | B1–B7 | 30m | ✅ |
| Landsat 8/9 | — | B8(全色), B9(卷云), B10–B11(热红外) | 15m/30m/100m | ❌ 跳过 |
| ASTER | vnir | B1, B2, B3N | 15m | ✅ |
| ASTER | swir | B4–B9 | 30m | ✅ |
| ASTER | — | B10–B14(热红外) | 90m | ❌ 跳过 |
| Sentinel-2 | vis10 | B2, B3, B4, B8 | 10m | ✅ |
| Sentinel-2 | red-edge | B5, B6, B7, B8A, B11, B12 | 20m | ✅ |
| Sentinel-2 | — | B1, B9, B10 | 60m | ❌ 跳过 |

## 架构变更：一个场景 → 多组输出

```
输入目录（场景）
  └── B1.tif … B14.tif

处理后输出
  └── 场景名/
      ├── ASTER_VNIR_corrected.tif   (15m, 3波段)
      └── ASTER_SWIR_corrected.tif   (30m, 6波段)
```

## 改动范围（仅 app.py）

### 1. 新增传感器分组配置常量 `SENSOR_GROUPS`

```python
SENSOR_GROUPS = {
    "landsat8": {
        "detect": lambda names: any(re.match(r'^B[1-9]$|^B1[01]$', n) for n in names),
        "groups": {
            "optical": {"bands": [1,2,3,4,5,6,7], "skip_res_filter": False},
        },
        "skip_bands": {8, 9, 10, 11},
    },
    "aster": {
        "detect": lambda names: "B3N" in names or (
            any(f"B{i}" in names for i in range(10,15))),
        "groups": {
            "VNIR": {"bands": ["1","2","3N"]},
            "SWIR": {"bands": ["4","5","6","7","8","9"]},
        },
        "skip_bands": {"10","11","12","13","14"},
    },
    "sentinel2": {
        "detect": lambda names: "B8A" in names,
        "groups": {
            "10m": {"bands": ["2","3","4","8"]},
            "20m": {"bands": ["5","6","7","8A","11","12"]},
        },
        "skip_bands": {"1","9","10"},
    },
}
```

传感器自动识别顺序：ASTER（有B3N）→ S2（有B8A）→ Landsat（其余）

### 2. 修改 `get_band_files()`

支持 B3N.tif、B8A.tif 这类非纯数字命名：

```python
BAND_FILE_PATTERN = re.compile(

)
```

返回 `dict[band_id → Path]` 而非列表，便于按分组取子集。

### 3. 修改 `read_image()` → `read_image_groups()`

返回分组列表而非单一数组：

大气校正中 `image[:7]` 的截断逻辑改为由分组配置决定，不再硬编码。

### 5. 保存路径规则

```

single 模式:        {output_dir}/{rel_path_stem}_corrected.tif
```
