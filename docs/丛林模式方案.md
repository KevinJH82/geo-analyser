# geo-analyser「丛林模式」蚀变找矿方案与实现

> 计划原件:`~/.claude/plans/geo-analyser-parsed-cookie.md`
> 本文为入库副本,并附**实现状态**与**使用方法**。

## Context（为什么做）

密林/丛林覆盖区,常规蚀变分析结构性失效:`remove_interference()` 把 NDVI>0.20 的像元判为植被置 NaN,丛林区(NDVI 0.7+)几乎整图被清空;且所有方法(SWIR 比值、Crosta PCA、EnMAP 吸收深度)都依赖地表岩石光谱直接出露,郁闭冠层下物理上看不到岩石。

**核心思路转变**:不再追求"看清岩石",而是更换探测代理——(1) 把植被从障碍变成找矿标志(地植物学胁迫),(2) 抢救部分出露像元(光谱解混 + 天然出露靶向),(3) 用雷达相干性绕过冠层勾勒控矿构造,(4) 直接蚀变失效时,用多源间接证据融合出"成矿有利度"。

**产出**:`/api/analyze_batch` 的 `jungle_mode` 开关(默认 False,零回归),密林区即便拿不到蚀变图也能输出多证据成矿有利度靶区。

## 四条路径与实现状态

### 路径1 地植物学胁迫探测 ✅ 已实现并验证
矿化/微渗漏使植物受胁迫 → 红边蓝移、叶绿素↓。在"有植被"像元上算活力(NDRE 默认),取负向 → 活力低=胁迫高=潜在矿化。
- `interference_removal.py`:`calc_ndre` / `calc_cire` / `calc_rep`(红边指数)
- `alteration_analysis.py`:`calc_veg_stress` + `analyze_single` 新增 `veg_stress` 方法分支(复用 `_threshold_anomaly` 稳健阈值)
- `alteration_db.py`:`jungle_veg_stress_target()` 通用胁迫目标(仅 Sentinel-2/EnMAP/PRISMA 红边传感器)
- 当前走 Sentinel-2 红边主路径;EnMAP/PRISMA 的红边胁迫为后续扩展(其 band_depth 仍正常)

### 路径2 多证据成矿预测融合 ✅ 已实现并验证(capstone)
直接蚀变失效时,用间接证据加权出成矿有利度。
- 新增 `prospectivity.py`:`fuse_evidence()` 融合 N 层证据(蚀变 E1 / 地植物胁迫 E2 / 构造 E3 / InSAR 形变 E4),任何层缺失自动忽略并对存在的层再归一权重(和=1);支持 `weighted_sum` 与 `fuzzy_gamma`
- `app.py`:`_compute_prospectivity()` 收集各层(复用 `structural_weighting` + `commons.insar_broker`),`api_analyze_batch` 在 `jungle_mode` 下输出 `prospectivity_*`;落盘见 `alteration_store.py`(新增 `__composite__prospectivity.tif/png`)
- **注**:地植物胁迫目标不计入"蚀变综合得分 score_map",避免与 E2 重复计数

### 路径3 SAR 穿透构造提取 ✅ 能力就绪(待数据接入)
- `insar_analysis.py`:`coherence_lineaments()` 对相干性图做多方向线性匹配滤波,提取冠层下低相干线性体(常追踪断裂/岩性接触带),含"全高相干→无线性体"退化保护
- **数据依赖**:当前 geo-insar broker 仅暴露 `velocity` / `deformation_evidence`,**无 coherence 产物**;函数已就绪,待 geo-insar 产出 `coherence.tif` 即可接入 prospectivity。后向散射/极化档需 geo-insar 新增 L 波段产物,不在本期

### 路径4 光谱解混 + 天然出露靶向 ✅ 已实现并验证
- 新增 `spectral_unmix.py`:`linear_unmix()`(NNLS 三端元{植被/岩石/阴影}解混,恢复岩石分量丰度)+ `detect_natural_exposures()`(BSI+低 NDVI 定位河岸/山脊/滑坡疤/采坑等裸露)
- `app.py`:`jungle_mode` 下报告天然出露占比(`composites[sensor].exposure`);解混作为按需重型能力保留
- 多时相出露合成需多季相数据装载扩展(`delivery_project.py`),作为后续(项目已有夏/冬两季数据,可后续启用)

## 验证结果（真实项目端到端)

测试项目:`布基纳法索6小区块-A_1780298223`(造山型金矿,西非植被覆盖区),Sentinel-2 冬季数据。

- **回归**:`jungle_mode=False` → grid 为空、无 prospectivity,现有行为零改变 ✓
- **丛林模式**:地植物胁迫走 Sentinel-2 出异常(0.08%);prospectivity 自动发现并融合 InSAR 形变证据(`used_layers=[veg_stress, deformation]`,权重再归一和=1),无构造数据时优雅降级;天然出露占比统计正常;结果落盘 ✓
- 各新模块均带合成数据单测(`python3 prospectivity.py` / `spectral_unmix.py`,以及 `interference_removal`/`insar_analysis` 的内联测试)全部 PASS

## 使用方法

```bash
POST /api/analyze_batch
{
  "project_name": "<项目名>",
  "deposit_type": "<矿床类型>",
  "roi_geojson":  { ... },
  "jungle_mode":  true,          # 开启丛林模式
  "use_structural_prior": true,  # 可选:叠加构造约束(E3)
  "threshold_method": "median_mad", "k": 2.0
}
```

返回新增:`grid` 中含"丛林地植物胁迫(红边)"条目;`composites[sensor].prospectivity`(证据层/权重/靶区像素/阈值)、`composites[sensor].prospectivity_overlay`(热力图)、`composites[sensor].exposure`(出露占比);manifest 落盘 `__composite__prospectivity.tif`。

需 Sentinel-2(红边波段)覆盖 ROI;缺失时胁迫目标记 note 并降级。
