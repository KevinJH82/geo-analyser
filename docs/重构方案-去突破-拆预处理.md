# geo-analyser 重构方案：移除尖点突破 + 拆出数据预处理

## Context（为什么 / 目标）

对 geo-analyser 做一次职责收窄的重构,使其**聚焦蚀变分析**:
1. **移除"尖点突破分析"**(聚类)的全部功能与内容;
2. **把"数据预处理"拆成独立子系统 `geo-preprocess`**,与 geo-downloader / geo-analyser / geo-stru 平级同风格;
3. geo-analyser 保留蚀变分析(含成矿预测融合、构造约束)及 **InSAR 形变监测**(用户确认保留)。

### 已确认决策
| 项 | 决策 |
|---|---|
| InSAR 独立监测(insar_analysis/insar_timeseries + /api/insar/*) | **保留在 geo-analyser**,本次不动 |
| 新预处理系统目录名 | **`geo-preprocess`** |
| 新系统前端 | **迁移完整前端**(大气/几何/干扰剔除三 tab),参照 geo-stru 范式 |
| 本次产出 | 先出方案,过目后再动代码 |

### 关键耦合结论(决定怎么切,已核验)
- 蚀变分析**不依赖**预处理流水线运行时产物:`analyze_batch` 经 `delivery_project.load_sensor_data` 直接读交付数据,自己做 ROI 掩膜 + 丛林红边胁迫。删预处理路由不影响蚀变。
- **唯一硬耦合**:`alteration_analysis.py:27` 与 `spectral_unmix.py` 从 `interference_removal.py` import 纯指数函数(`calc_ndvi/calc_ndre/calc_cire/calc_rep/calc_bsi/_safe_divide`)。→ 抽到 `commons/spectral_indices.py` 共享(见 Part 2-C)。
- `insar_analysis.py`/`insar_timeseries.py` 仅被 `/api/insar/*`(app.py:3449/3450/3570)使用,**与蚀变解耦** → 按决策保留不动。
- `structural_weighting.py` 被蚀变成矿预测**深度依赖**(app.py 多处 + `_compute_prospectivity`) → **保留**。
- 成矿预测的 InSAR 证据来自 `commons.insar_broker`(读 geo-insar 形变),非 `insar_analysis.py` → 不受影响。
